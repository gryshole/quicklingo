# Security Policy

## Supported Versions

Security fixes are applied to the latest release on the `main` branch.

| Version | Supported |
| ------- | --------- |
| latest  | yes       |

## Reporting a Vulnerability

If you discover a security issue in QuickLingo, please report it privately:

1. Open a [GitHub Security Advisory](https://github.com/gryshole/quicklingo/security/advisories/new) (preferred), or
2. Email the maintainer via the contact address on their [GitHub profile](https://github.com/gryshole).

Please include:

- A description of the issue and potential impact
- Steps to reproduce
- Affected version or commit (if known)

Do not open a public issue for undisclosed security vulnerabilities.

We aim to acknowledge reports within a few days and will work with you on a fix and coordinated disclosure when appropriate.

## Scope

This policy covers the QuickLingo application source code in this repository. It does not cover third-party API providers (Groq, Google Gemini, OpenRouter, etc.) — report issues with those services directly to the respective provider.
